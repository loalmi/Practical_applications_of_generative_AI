Добавлен инструмент для сравнения метрик в двух периодах.
Примеры запросов:
- "Во сколько раз вырос курс евро с января 2020 по январь 2026?"
- "Сравни инфляцию в декабре 2020 и декабре 2025"

## 3. Диагностика трёх ошибок

### Ошибка 1: Нет данных в CSV (Q6)
- **Тип:** Инструмент упал
- **Трасса:** `get_inflation(2020-12) → error: нет данных ИПЦ на 2020-12`
- **Причина:** В CSV нет данных за декабрь 2020
- **Что бы исправили:** Проверить наличие данных или использовать `compare_periods` только для доступных дат

### Ошибка 2: Ожидаемое слово не найдено в ответе (Q7)
- **Тип:** Галлюцинация/неточность в финальном ответе
- **Трасса:** Ответ не содержит `"2023-03"`
- **Причина:** Агент написал "март 2023" вместо точного формата
- **Что бы исправили:** Ужесточить промпт для точного вывода дат

### Ошибка 3: Нет ожидаемого инструмента в трассе (Q10)
- **Тип:** Неправильный порядок вызовов
- **Трасса:** Агент не вызвал `compare_periods`
- **Причина:** Агент не понял, что для сравнения можно использовать `compare_periods`
- **Что бы исправили:** Улучшить описание инструмента в `schemas.py`

## 4. Вывод

Самый хрупкий элемент системы — **зависимость от наличия данных в CSV** и **точность описания инструментов в схеме**. Если данных нет или описание инструмента недостаточно чёткое, агент либо падает, либо выбирает неправильный путь.
